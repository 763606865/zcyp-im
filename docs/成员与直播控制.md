# 成员与直播控制

以下接口都需要：

```http
Authorization: Bearer <token>
```

## 1. 获取成员列表

- 方法：`GET`
- 路径：`/im/conversations/:conversation_no/members`

## 2. 添加成员

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/members`

### 请求体

```json
{
  "member_user_ids": ["u_1002", "u_1003"]
}
```

### 说明

- 只有 `owner` 可调用
- 添加成员时要求用户状态为 `active`

## 3. 移除成员

- 方法：`DELETE`
- 路径：`/im/conversations/:conversation_no/members/:member_user_id`

### 说明

- 只有 `owner` 可调用
- 不能移除 `owner`

## 4. 加入会话

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/join`

### 说明

- 仅 `chatroom` / `live_room` 支持
- 被封禁用户不能加入

## 5. 退出会话

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/leave`

### 说明

- 仅 `chatroom` / `live_room` 支持
- `owner` 不能主动退出

## 6. 封禁与解封

- `POST /im/conversations/:conversation_no/members/:member_user_id/ban`
- `POST /im/conversations/:conversation_no/members/:member_user_id/unban`

### 说明

- `owner` / `admin` 可调用
- 不能封禁 `owner`
- 解封后成员状态会变为 `left`

## 7. 调整角色

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/members/:member_user_id/role`

### 请求体

```json
{
  "role": "speaker"
}
```

### 角色规则

`live_room` 支持：

- `admin`
- `speaker`
- `audience`

`group` / `chatroom` 支持：

- `admin`
- `member`

`single` 支持：

- `member`

## 8. 禁言与解禁

- `POST /im/conversations/:conversation_no/members/:member_user_id/mute`
- `POST /im/conversations/:conversation_no/members/:member_user_id/unmute`

### 禁言请求体

```json
{
  "minutes": 30
}
```

### 说明

- `owner` / `admin` 可调用
- 不能禁言 `owner`

## 9. 全员禁言

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/all-muted`

### 请求体

```json
{
  "enabled": true
}
```

### 说明

- `owner` / `admin` 可调用
- 开启后，除 `owner` / `admin` 以外成员不能发言

## 10. 内容审核开关

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/review`

### 请求体

```json
{
  "enabled": true
}
```

### 说明

- `owner` / `admin` 可调用
- 开启后，消息会命中本地配置的敏感词检查

## 11. 麦位状态

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/members/:member_user_id/mic`

### 请求体

```json
{
  "mic_status": "on"
}
```

### 说明

- 仅 `live_room` 支持
- 仅 `owner` / `admin` 可操作
- 可选值：
  - `on`
  - `off`
- `speaker` 必须 `mic_status=on` 才能发言
