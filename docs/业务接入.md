# 业务接入

本文档描述面向业务系统的 `/api` 前缀接口。

这组接口用于业务侧服务接入 IM，例如业务注册用户、更新昵称头像、查询用户信息、签发 IM 连接 token。

## 1. 鉴权方式

所有 `/api/*` 接口都需要在请求头中传递：

```http
X-App-Code: app_xxxxxxxx
X-App-Key: your_app_key
Content-Type: application/json
```

说明：

- `X-App-Code`：应用编码
- `X-App-Key`：应用密钥
- 这组接口适合业务后端服务调用，不建议直接暴露给公开客户端

## 2. 创建或更新用户

- 方法：`POST`
- 路径：`/api/users`

请求体：

```json
{
  "external_user_id": "u_1001",
  "nickname": "张三",
  "avatar_url": "https://example.com/avatar.png"
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "app_id": 1,
    "external_user_id": "u_1001",
    "nickname": "张三",
    "avatar_url": "https://example.com/avatar.png",
    "status": "active",
    "created_at": "2026-07-09T11:00:00+08:00",
    "updated_at": "2026-07-09T11:00:00+08:00"
  },
  "message": "",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 2
  }
}
```

说明：

- 当用户不存在时会创建
- 当用户已存在时会更新昵称和头像
- 已有用户状态会被保留，不会因为更新资料而自动解除禁用

## 3. 查询用户

- 方法：`GET`
- 路径：`/api/users/:external_user_id`

成功响应：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "app_id": 1,
    "external_user_id": "u_1001",
    "nickname": "张三",
    "avatar_url": "https://example.com/avatar.png",
    "status": "active",
    "created_at": "2026-07-09T11:00:00+08:00",
    "updated_at": "2026-07-09T11:00:00+08:00"
  },
  "message": "",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 1
  }
}
```

## 4. 更新用户资料

- 方法：`PUT`
- 路径：`/api/users/:external_user_id`

请求体：

```json
{
  "nickname": "李四",
  "avatar_url": "https://example.com/avatar-new.png"
}
```

说明：

- 字段按需传递
- 未传字段会保留原值
- 该接口只更新用户资料，不修改用户状态

## 5. 签发 IM Token

- 方法：`POST`
- 路径：`/api/users/:external_user_id/token`

成功响应：

```json
{
  "code": 200,
  "data": {
    "app_code": "app_xxxxxxxx",
    "user_id": "u_1001",
    "token": "jwt_token",
    "expires_at": "2026-07-10T11:00:00+08:00"
  },
  "message": "",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 1
  }
}
```

说明：

- 只有用户存在且状态为 `active` 时才能签发 token
- 返回的 token 可直接用于：
  - HTTP IM 接口：`Authorization: Bearer <token>`
  - WebSocket：`/im/connect?token=<token>`

## 6. 常见错误

应用鉴权失败：

```json
{
  "code": 401,
  "data": null,
  "message": "invalid app credentials",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 0
  }
}
```

用户不存在：

```json
{
  "code": 404,
  "data": null,
  "message": "user not found",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 0
  }
}
```

用户已禁用：

```json
{
  "code": 403,
  "data": null,
  "message": "user disabled",
  "meta": {
    "timestamp": "2026-07-09T11:00:00+08:00",
    "response_time": 0
  }
}
```
