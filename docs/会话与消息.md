# 会话与消息

以下接口都需要：

```http
Authorization: Bearer <token>
```

## 1. 创建会话

- 方法：`POST`
- 路径：`/im/conversations`

### 请求体

```json
{
  "conversation_key": "order:1001:service",
  "type": "group",
  "scene": "",
  "subject": "项目群",
  "member_user_ids": ["u_1002", "u_1003"]
}
```

### 会话类型

- `single`
- `group`
- `chatroom`
- `live_room`

### 规则

- 创建者会自动成为 `owner`
- `conversation_key` 可选；同一应用下传入相同 key 时会直接复用已存在会话
- `single`：`member_user_ids` 必须且只能有 1 个用户
- `group`：允许初始化成员
- `chatroom` / `live_room`：不允许初始化成员
- `scene = system`：使用业务 `/api` 创建 `single` 会话，由系统账号作为 owner、接收用户作为唯一成员；用户侧只读

### 成功响应

```json
{
  "id": 1,
  "conversation_no": "conv_xxxxxxxx",
  "conversation_key": "order:1001:service",
  "app_id": 1,
  "type": "group",
  "subject": "项目群",
  "owner_user_id": "u_1001",
  "all_muted": false,
  "require_review": false,
  "created_at": "2026-07-08T15:00:00+08:00",
  "updated_at": "2026-07-08T15:00:00+08:00"
}
```

## 2. 发送消息

- 方法：`POST`
- 路径：`/im/conversations/:conversation_no/messages`

### 请求体

```json
{
  "message_type": "text",
  "client_msg_id": "client_001",
  "content": {
    "text": "hello"
  }
}
```

### 说明

- `content` 统一为结构化 JSON 对象
- `text` 消息建议格式：`{"text":"hello"}`
- 业务卡片消息可直接传结构化对象，例如 `message_type = "biz_card"`
- 当前 `live_room` 只允许 `text` 消息
- `scene = system` 的会话仅允许业务 `/api` 以系统账号发送 `system_notice`，用户侧接口和 WebSocket 不能发送
- 会校验发言权限、禁言状态、全员禁言、内容审核开关

### 卡片消息示例

```json
{
  "message_type": "biz_card",
  "client_msg_id": "client_002",
  "content": {
    "card_type": "oa_approval",
    "title": "审批申请",
    "summary": "张三提交了请假申请，等待你审批",
    "status": "pending",
    "action_url": "/oa/approvals/12345",
    "biz_id": "approval_12345"
  }
}
```

### 成功响应

```json
{
  "id": 1,
  "message_no": "msg_xxxxxxxx",
  "app_id": 1,
  "conversation_id": 1,
  "sender_user_id": "u_1001",
  "message_type": "text",
  "client_msg_id": "client_001",
  "content": {
    "type": "text",
    "text": "hello"
  },
  "created_at": "2026-07-08T15:20:00+08:00"
}
```

### 常见错误

```json
{
  "error": "conversation speak not allowed"
}
```

```json
{
  "error": "conversation member muted"
}
```

```json
{
  "error": "conversation review rejected"
}
```

## 3. 获取历史消息

- 方法：`GET`
- 路径：`/im/conversations/:conversation_no/messages`

### 查询参数

- `limit`：可选，默认 `50`，范围 `1-200`

### 说明

- 只有会话成员可查询
